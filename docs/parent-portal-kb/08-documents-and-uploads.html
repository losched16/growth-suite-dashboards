Documents & file uploads

Article 8 of 12 · Parent Portal Knowledge Base

Not every document fits inside a form. The Documents section is the catch-all for files parents send to your school: immunization records, court orders, insurance cards, doctor’s notes, custom paperwork. This article covers how it works for both sides.

What parents see

The Documents page (sometimes shown alongside Forms in the navigation) has two parts:

1. The upload form

  • File chooser — picks a file from their device.
  • Display name (optional) — a friendly label for the file; defaults to the filename if blank.
  • For which student (optional) — lets the parent tag the document to a specific child.
  • For which form (optional) — if your school has a tracker form set up that this document fulfills, the parent can link it.
  • Notes (optional) — free-form note to the school office.

2. Their upload history

Below the form, every file the family has previously uploaded, with download links, a status badge, and a delete button.

What kinds of files are accepted

The portal accepts most common document types:

  • PDFs (most common)
  • Images: JPG, PNG, HEIC, WEBP
  • Word documents (DOC, DOCX)
  • Excel spreadsheets (XLS, XLSX)
  • Plain text (TXT)

Max file size is 10 MB. For larger files, parents should email the office directly — the upload field will refuse oversized files with a clear error message.

What happens when a parent uploads a file

  1. The file is stored securely in Growth Suite, attached to the family’s record.
  2. An entry appears in the admin Family Uploads dashboard.
  3. The file is also pushed to the parent’s contact thread in your admin Conversations area — so it shows up as part of their conversation history with the school.
  4. The admin row shows the file name, who uploaded it, when, and the optional note.

Statuses parents see

Each upload in their history shows a small status badge:

BadgeWhat it means
sending…Just uploaded; we’re pushing it to the school’s side.
sent to schoolThe school has received it on their end. Visible in their admin dashboard.
acknowledgedA staff member has opened or processed it. (Only shown if your school’s workflow updates this status.)
syncing…Something hiccupped during the push to the admin side. We retry automatically.

What school staff see

On the admin side, the Family Uploads page lists every upload across every family, sortable and searchable. Each row shows:

  • Family name and parent who uploaded
  • File name, size, mime type
  • Upload timestamp
  • Any note the parent added
  • Tags: which student, which form (if specified)
  • An Acknowledge button so staff can mark they’ve seen it (which updates the badge on the parent side)
  • A Download link

When to use Documents vs Forms file uploads

Some forms include a file upload field too (e.g. an Immunization Exemption form that asks for the signed PDF). The rule of thumb:

  • Use a Form file upload when the file is part of a specific structured submission — e.g. it’s required to complete a particular form.
  • Use the Documents section for ad-hoc files that don’t belong to any specific form — a court order, an insurance card update, a doctor’s note for a missed day.

Privacy and security

  • Uploaded files are stored encrypted at rest.
  • Only authenticated parents in the same family can download a file.
  • School staff with admin access can download any upload from their school.
  • Files are never shared with other schools, other families, or any third party.

Common use cases for Documents

  • Annual immunization record (state-mandated PDFs, image of the yellow card)
  • Custody / court documentation
  • Updated insurance card
  • Doctor’s note explaining an absence
  • School-requested forms that aren’t in the Portal Forms system yet
  • IEP, 504 plans, or learning accommodation paperwork

Tip: Encourage parents to add a quick note when they upload. “Updated immunization card — got booster on Sept 12” is much more useful than a filename like IMG_4837.jpg.
